Cloudflare Access（Zero Trust）で保護された環境において、PWAのService WorkerがHTMLをキャッシュしてしまうことで、認証切れ時にJSファイルがCORSエラー（Failed to fetch）を起こす問題を修正してください。

【対象ファイル: `public/sw.js`】
以下の要件を満たすように書き換えてください。

1. キャッシュのクリアを強制するため、上部の `CACHE_NAME` を `'rss-reader-v2'` に変更してください。
2. `fetch` イベントの処理を以下のハイブリッド戦略（HTMLのみNetwork-First）に変更してください。

// キャッシュ優先（Cache-First）でネットワーク通信を傍受
self.addEventListener('fetch', (event) => {
  // 1. APIリクエストは常にネットワークから
  if (event.request.url.includes('/api/')) {
    return;
  }

  // 2. 画面の遷移（HTML）はネットワーク優先
  // 認証切れの際にCloudflare Accessのログイン画面へ正しくリダイレクトさせるため
  if (event.request.mode === 'navigate') {
    event.respondWith(
      fetch(event.request).catch(() => {
        return caches.match(event.request);
      })
    );
    return;
  }

  // 3. その他の静的アセット（JS, CSS, 画像）はキャッシュ優先
  event.respondWith(
    caches.match(event.request).then((cachedResponse) => {
      if (cachedResponse) {
        return cachedResponse;
      }
      return fetch(event.request);
    })
  );
});
