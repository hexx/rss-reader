スマートフォンでホーム画面に追加（PWA）した際や、ブラウザのタブ上でアイコンが正しく表示されない問題を解決し、同時にアプリを完全にPWA（Progressive Web App）として動作させるための修正を行ってください。

【1. HTMLのヘッド要素の修正 (`index.html`)】
- ブラウザのタブ用の Favicon と、iOS（Safari）でホーム画面に追加した際用のアイコン（Apple Touch Icon）のメタタグが不足しています。
- `index.html` の `<head>` 内に、以下のタグを正確に追記してください。
  <link rel="icon" type="image/png" href="/icon-192.png" sizes="192x192" />
  <link rel="icon" type="image/x-icon" href="/favicon.ico" />
  <link rel="apple-touch-icon" href="/icon-192.png" />

【2. Webアプリマニフェストの拡張 (`public/manifest.json`)】
- 現在 `icons` フィールドが不足しています。PWAとしてホーム画面に追加した際にアイコンが適用されるよう、以下の定義を `public/manifest.json` に追加してください。
  "icons": [
    {
      "src": "/icon-192.png",
      "sizes": "192x192",
      "type": "image/png",
      "purpose": "any maskable"
    },
    {
      "src": "/icon-512.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "any maskable"
    }
  ]

【3. サービスワーカーの作成 (`public/sw.js`)】
- ブラウザにPWAとしてインストール可能と認識させるため、最小限のサービスワーカーファイル `public/sw.js` を新規作成してください。中身は静的アセットのキャッシュ、または以下のようなシンプルなフェッチリスナーを持つ最小構成で構いません。
  self.addEventListener('fetch', (event) => {
    // 最小限のPWA要件を満たすためのダミーフェッチハンドラー
  });

【4. サービスワーカーの登録 (`src/client/main.tsx`)】
- アプリ起動時に上記で作成したサービスワーカーをブラウザに登録するロジックを `src/client/main.tsx` の末尾に追記してください。
  if ('serviceWorker' in navigator) {
    window.addEventListener('load', () => {
      navigator.serviceWorker.register('/sw.js').catch((err) => console.error('SW register failed:', err));
    });
  }

【5. アイコン画像アセットの作成と配置】
- アプリのテーマカラーである「#4f46e5 (インディゴブルー)」をベースにした、RSSリーダーにふさわしいシンプルなアイコン画像を自動生成し、以下のパスに適切なサイズで配置してください。
  - `public/favicon.ico`
  - `public/icon-192.png` (192x192px PNG)
  - `public/icon-512.png` (512x512px PNG)

【6. 静的アセットのビルド検証】
- 修正完了後、`npm run build` を実行してフロントエンドのビルド（`dist-client` への出力）を行い、新しく追加した画像や `sw.js` が、バックエンド（Wrangler / ASSETS binding）経由で正常にアクセス可能であることを確認してください。
