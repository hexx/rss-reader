# pull-request

## いつ使うべきか (When to Use)
- ユーザーから「プルリクエストを作成して」「PRを出して」と指示されたとき。

## 前提条件と確認ポイント (Pre-checks)
コマンドを実行する前に、必ず以下の項目を順番に確認してください。

1. **すべてのテキストを日本語にすること**
   - 生成するPRタイトル、PR概要（Description）、およびユーザーへの進捗報告はすべて日本語で記述してください。
2. **既存のプルリクエストの状態を確認すること**
   - 同一ブランチからの以前のプルリクエストが、すでにマージまたはクローズされていないか事前に確認してください。
3. **コンフリクトがない状態にすること**
   - ターゲットブランチ（`main`）の最新コミットをローカルに一度取り込み、コンフリクトが発生しないこと（すぐにマージできる状態であること）を事前に確認してください。

## 処理フロー (Quick Start)
1. `git fetch origin main` および `git merge-base` 等を用いて、`main` ブランチとの競合（コンフリクト）がないかチェックする。
2. `git diff` を実行して現在のコード変更内容をすべて把握する。
3. 変更内容に基づいて、短くわかりやすい日本語の「PRタイトル」を作成する。
4. 変更の目的と要約をまとめた日本語の「PR概要（Description）」を作成する。
5. 以下のGitHub CLI（gh）コマンドを実行して、`main` ブランチに向けてプルリクエストを作成する。
   `gh pr create --base main --title "生成した日本語タイトル" --body "生成した日本語概要"`
